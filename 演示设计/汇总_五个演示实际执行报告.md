-    if (radio_recv_fnc(dummy_buffer, &rx_time) == SRSRAN_SUCCESS) {

+      Warning("SYNC:  Radio receive failed while in IDLE_RX");

## 演示五：Multi-Agent 并行 Review

### 执行方式

应用演示一的 patch 后，并行启动 3 个 SubAgent 审查同一份代码变更：

| Agent | 角色 | 审查维度 | 耗时 |
|-------|------|---------|------|
| 安全审查 | security-reviewer | CWE/MISRA/值域冲突/错误路径 | ~125s |
| 代码质量 | code-quality-reviewer | 命名/编号/测试/日志 | ~115s |
| 架构兼容性 | architecture-reviewer | 调用链/ABI/跨模块/文档 | ~122s |

### 自定义 Agent 文件

```
srsRAN_4G/.claude/agents/
├── security-reviewer.md        ← tools: Read,Grep,Glob,Bash（只读）
├── code-quality-reviewer.md
└── architecture-reviewer.md
```

### 核心发现（三方共识）

**所有 3 个 Agent 独立发现了同一个核心问题**：

> C 层 `receive_samples()`（ue_sync.c:722）将所有负返回值统一映射为 `SRSRAN_ERROR`(-1)，导致新增的 `SRSRAN_ERROR_RADIO_RECV_FAIL`(-9) 在 CAMPING 状态下被吞没。新错误码的区分价值仅限于 IDLE 路径。

| 级别 | 安全 Agent | 质量 Agent | 架构 Agent |
|------|-----------|-----------|-----------|
| ue_sync.c 吞没错误码 | Critical | Critical | Medium |
| run_idle_state 缺兜底分支 | High | High | Medium |
| NR 路径未同步更新 | Medium | High | Low |
| 缺少单元测试 | - | Low | - |
| CLAUDE.md 描述过时 | - | Medium | - |

**教学价值**：
- 三个独立视角发现同一核心问题 → **上下文隔离带来真正独立的视角**
- 安全/质量/架构各有侧重但互补 → **多角度审查的实际价值**
- AI Review 发现了人类可能忽略的问题 → **Multi-Agent 不是噱头，是生产力**
