## 二、各演示摘要

### 演示一：Agent Loop 实战 — 新增错误码

**指令**：
> "srsRAN_4G 的 sync::radio_recv_fnc 在射频接收失败时返回通用的 SRSRAN_ERROR，请新增一个专用错误码 SRSRAN_ERROR_RADIO_RECV_FAIL 并让相关调用点正确处理它。"

**关键数据**（实测 2026-05-28）：25 轮 / 257 秒 / $0.58 / 改 2 文件 / 缓存 95%+

**核心讲点**：
- 前 5 轮全在"阅读理解"，后 3 轮才"动手改"
- 只改 2 个文件（ue_sync.c 的 `< 0` 检查天然兼容 -9，不改才对）
- 最后跑 cmake --build 做自我校验

### 演示二：Langfuse 可观测性 — 执行轨迹

**要点**：
- 展示 Trace 嵌套 Span 结构（每轮 Loop 一个 span）
- **重要**：CLAUDE.md 在 **User Message** 中而非 System Prompt 中（命中 prompt cache）
- Token 缓存命中率 95% 可视化

### 演示三：CLAUDE.md 四层实战

**要点**：
- 推荐"打开预写文件逐段讲解"（3min 手写太紧）
- 项目级 ~40 行：模块结构表 + 编码约定 + Skill 路由
- 企业级：MISRA/gets()/IMSI 红线；本地级：JIRA+交叉编译便签

### 演示四：Stop Hook — C 代码语法检查门禁

**要点**：
- 脚本：git diff 找 C/C++ 文件 → gcc/g++ -fsyntax-only → exit 2 阻止 → Claude 自动修复
- C 文件用 `-std=c11`，C++ 用 `-std=c++14`（Review 修正）
- 防死循环：stop_hook_active 字段检查
- 需课前验证 include path（高概率失败风险已加预案）

### 演示五：Multi-Agent — 调研→Review→迭代

**四步流程**：
1. 调研 Agent 出设计方案
2. 并行 3 个 Review Agent（安全/质量/架构兼容性）
3. 汇总 Review 报告（Critical/High/Medium/Low）
4. 基于报告迭代

**核心风险**：Claude 可能不启动并行 SubAgent → 需要显式指令 + PPT 备份
