## 二、跨视角共识（多个视角同时指出）

### 共识 1：CLAUDE.md 加载位置描述自相矛盾 🔴

- **AI 专家**：演示二 Step 3 说"CLAUDE.md 在 User Message 中"，但第七节表格又写"System Prompt 中的位置"，两处矛盾
- **课程设计**：正文 §3.2 明确说走 User Content 路径，如果 Langfuse 展示与此不符，教学论点崩塌
- **修复**：统一描述，课前实际截取 Langfuse trace 验证

### 共识 2：Stop Hook 的 include path 问题 🟡

- **C/嵌入式专家**：`gcc -fsyntax-only` 遇到 `#include <fftw3.h>` 就会报错，只能检查自有头文件闭合的简单文件
- **学员视角**：内部项目依赖大量内部头文件和交叉编译工具链，方案不够落地
- **修复**：补充提及 `compile_commands.json` + `clang-tidy` 方案

### 共识 3：演示五现场跑通风险高 🟡

- **课程设计**：弹性区间 8-15 分钟过大，超时概率高
- **学员视角**：3 个并行 API 请求万一限流就尴尬
- **AI 专家**：并行执行依赖 Claude Code 版本，可能顺序执行
- **修复**：预跑报告作默认展示素材，现场跑作加分项

### 共识 4：security-reviewer 的 Bash 工具 ≠ 物理只读 🟡

- **AI 专家**：Bash 可执行 sed/echo > 等写操作，"物理上做不到"说法不严格
- **修复**：去掉 Bash 或改措辞为"prompt 约束"
