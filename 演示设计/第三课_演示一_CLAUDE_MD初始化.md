## 一、演示目标

让学员看到完整的 CLAUDE.md 生成工作流：（1）一条 prompt 让 Claude 扫描项目并生成结构化 CLAUDE.md；（2）Review 生成结果确保行数合规；（3）超出部分渐进式披露到 docs/ 子文档。同时保留现有 CLAUDE.md 内容。

**核心教学点**：
1. AI 80% + 人工 20%——AI 能自动提取项目元信息，但填不出历史决策
2. CLAUDE.md 有房租——400 行硬上限，超 800 行 Skill 调用率低 40%
3. 渐进式披露——核心信息在 CLAUDE.md，详细内容通过链接指向 docs/ 子文档
4. Review 是流程一部分——生成后必须检查行数和内容质量

## 三、演示指令

### Step 1：生成 CLAUDE.md（完整版 Prompt）

```
读一下这个项目的 README、顶层目录结构、CMakeLists.txt 和主要源文件，
生成 CLAUDE.md 初稿。要求：

1. 项目定位：一句话说明项目是什么、解决什么问题
2. 核心架构：
   - 对外接口和系统实体（如独立的可执行程序、库、协议实体等）
   - 核心模块拓扑和职责
   - 关键依赖和技术栈
3. 关键模块：每个顶级目录的职责和核心文件
4. 构建命令：完整的构建、测试、运行命令
5. 命名约定：函数、宏、文件的命名规范
6. "禁区"章节留空，写"待人工补充"
7. "历史包袱"章节留空，写"待人工补充"

约束：
- 总行数控制在 300 行以内（硬上限 400 行）
- 架构图、接口清单、数据模型的详细内容不要写进来，
  用 `→ 详见 docs/xxx` 链接指向 docs/ 子文档
- 保留项目中已有的 CLAUDE.md 内容（如果存在的话），
  在此基础上补充和优化
```

**讲解**："注意这个 prompt——不限定项目类型。你们的项目不一定是协议栈，不一定是 C，这个 prompt 适用于任何项目。"

### Step 2：Review 生成结果（Prompt）

```
检查刚才生成的 CLAUDE.md：
1. 统计总行数，报告是否超过 400 行
2. 如果超过 400 行：
   - 找出可以提取到 docs/ 子文档的章节（如详细构建说明、完整接口列表等）
   - 把这些内容移到对应的 docs/ 文件中
   - 在 CLAUDE.md 中用 `→ 详见 docs/xxx.md` 链接替代
   - 重新统计行数，确保 ≤ 400 行
3. 检查"禁区"和"历史包袱"章节是否留空（不能让 AI 编造）
4. 检查架构描述是否准确反映项目实际结构
```

**讲解**："Review 不是可选的——生成后必须检查。行数超了就做渐进式披露：详细内容移到 docs/，CLAUDE.md 只放链接。"

### Step 3：渐进式披露补充（如需要）

如果 Review 发现需要分拆，展示产出结构：

```
CLAUDE.md（≤ 400 行，核心信息）
  ├── "→ 详见 docs/build-guide.md"
  ├── "→ 详见 docs/architecture.md"
  └── "→ 详见 docs/naming-conventions.md"

docs/（详细内容）
  ├── build-guide.md        ← 完整构建、测试、部署说明
  ├── architecture.md       ← 详细架构设计和模块关系
  └── naming-conventions.md ← 完整命名规范和代码风格
```

## 六、关键讲解点

### 6.1 AI 80% + 人工 20%

> "AI 填不出禁区和历史包袱——**因为 AI 不知道你们的历史决策。** 这 20% 人工补充，决定了 AI 不会碰不该碰的代码。"

### 6.2 400 行硬上限 + 渐进式披露

> "CLAUDE.md 的每一行都有房租——400 行硬上限。超了不是删——是做渐进式披露：详细内容移到 docs/，CLAUDE.md 只放链接。跟你们 Simba 项目做头文件拆分一个道理——主文件放声明，详细实现放子文件。"

### 6.3 Review 是流程一部分

> "生成完不是结束。Review 行数是否超限、内容是否准确、禁区是否留空——这些都需要检查。参考 Robert 方法论：每一步跑完自己 review 输出质量。"

| 风险 | 概率 | 应对 |
|------|------|------|
| CC 生成超过 400 行 | 中 | 正好演示渐进式披露——移到 docs/ |
| CC 没有留空禁区 | 低 | 手动指出"AI 猜了一些禁区，但不一定对——必须人工校验" |

| CC 未保留现有内容 | 低 | 说明"prompt 要求保留——实际使用时注意" |
