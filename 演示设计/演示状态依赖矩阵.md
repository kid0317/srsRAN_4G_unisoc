# 演示状态依赖矩阵

**用途**：确保任意一个演示失败不影响后续演示的独立执行。

---

## 依赖关系

```
演示一 ──→ 演示二（Langfuse trace 来自演示一的执行）
  │
  └──→ 演示五（Review 对象是演示一的代码变更）

演示三、演示四：独立，无前置依赖
```

## 每个演示的独立运行条件

| 演示 | 前置状态 | 独立运行方案（前序失败时） | 恢复操作 |
|------|---------|------------------------|---------|
| 一 | git 干净状态 | 直接跑（无依赖） | `git restore lib/include/srsran/config.h srsue/src/phy/sync.cc` |
| 二 | Langfuse 中有 trace | **课前预跑**一次演示一产生 trace；失败时用 PPT 截图 | 无需恢复 |
| 三 | 无 | 完全独立，打开预写好的文件展示 | 如需重跑演示一，先 `rm CLAUDE.md` |
| 四 | git 干净状态 + hooks/ 存在 | 完全独立，可单独运行 | `git restore` 恢复修改的文件 |
| 五 | 演示一的代码变更已应用 | `git apply 演示一_实际执行.patch` 快速应用；失败时用预跑好的 Review 报告 PPT | `git restore` 恢复 |

## 课堂演示顺序建议

1. **课前准备**：预跑演示一（headless 模式），确保 Langfuse 有 trace 数据
2. **课堂演示一**：现场跑（如失败切 PPT + patch 数据）
3. **课堂演示二**：打开 Langfuse 展示（如无数据切预存截图）
4. **课堂演示三**：打开预写文件展示（完全独立）
5. **课堂演示四**：现场给 Claude 发指令触发 Hook（如失败切 PPT）
6. **课堂演示五**：展示预跑好的 Review 报告（默认方案）；如时间充裕现场跑 1 个 Agent

## 关键物料检查清单

| 物料 | 路径 | 用途 |
|------|------|------|
| 演示一 Patch | `演示设计/演示一_实际执行.patch` | 快速应用代码变更 |
| 演示五 Review 报告 | `演示设计/演示五_MultiAgent_Review结果.md` | PPT 备份素材 |
| Langfuse 截图 | 课前截取 | 演示二 PPT 备份 |
| 四层 CLAUDE.md 文件 | srsRAN_4G/ 下 + 演示设计/ 下 | 演示三直接展示 |
